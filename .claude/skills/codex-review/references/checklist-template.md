# 审查清单模板

本文件定义 Codex 审查的标准检查项。
项目可创建 `project-checklist.md` 添加项目特定检查项。

---

## 通用检查项

### 代码质量
- [ ] 代码符合项目编码规范
- [ ] 无明显的 bug 或逻辑错误
- [ ] 错误处理完善（try-catch、错误提示）
- [ ] 无未使用的导入或变量

### 文档一致性
- [ ] 代码变更有对应的文档更新
- [ ] 文档描述与代码实现一致
- [ ] 版本号正确更新（代码版本 = 文档版本）
- [ ] 变更记录已更新

### 类型安全
- [ ] TypeScript 类型定义完整
- [ ] 无 `any` 类型滥用
- [ ] 接口定义清晰
- [ ] 泛型使用得当

---

## 前端特定检查

### UI 组件
- [ ] 组件职责单一（SRP）
- [ ] Props 类型定义完整
- [ ] 无硬编码的样式/文本
- [ ] 支持主题/暗色模式（如适用）
- [ ] 可访问性（a11y）考虑

### 状态管理
- [ ] 状态更新逻辑清晰
- [ ] 无不必要的重渲染
- [ ] 副作用处理得当（useEffect 依赖正确）
- [ ] 异步操作有加载/错误状态

### 表格/列表组件
- [ ] 列配置与文档一致
- [ ] 排序/筛选逻辑正确
- [ ] 分页功能正常
- [ ] 空状态处理

---

## 配置检查

### 版本号
- [ ] 代码中的配置版本号已更新
- [ ] 文档中的版本号已同步
- [ ] 版本号格式统一（v0.x.x 或 v1.x.x）

### 常量配置
- [ ] 新增配置项有注释说明
- [ ] 配置值合理（无硬编码的敏感信息）
- [ ] 默认值设置正确

---

## 审查结果格式

审查完成后，使用以下格式输出结果：

```markdown
## Codex 审核结果

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 代码质量 | ✅/⚠️/❌ | 具体说明 |
| 文档一致性 | ✅/⚠️/❌ | 具体说明 |
| 类型安全 | ✅/⚠️/❌ | 具体说明 |
| UI 组件 | ✅/⚠️/❌ | 具体说明（如适用） |
| 配置检查 | ✅/⚠️/❌ | 具体说明（如适用） |

### 问题清单
- [🔴 严重] 问题描述
- [🟡 警告] 问题描述
- [🟢 建议] 问题描述

### 修复建议
1. 建议内容
2. 建议内容
```

---

## 状态说明

| 状态 | 含义 | 行动 |
|------|------|------|
| ✅ | 通过 | 无需操作 |
| ⚠️ | 有警告 | 建议改进，非阻塞 |
| ❌ | 未通过 | 必须修复后再提交 |

---

## 严重度说明

| 级别 | 标记 | 说明 |
|------|------|------|
| 严重 | 🔴 | 必须立即修复，阻塞发布 |
| 警告 | 🟡 | 应该修复，不阻塞发布 |
| 建议 | 🟢 | 可选改进，提升代码质量 |
