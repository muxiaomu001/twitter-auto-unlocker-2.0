# 默认检测模式

以下是 Codex Review 的默认文件检测模式，适用于大多数前端项目。
项目可创建 `project-patterns.md` 覆盖这些默认值。

---

## 文档文件（触发文档审查）

| 模式 | 说明 |
|------|------|
| `docs/**/*.md` | 所有文档目录下的 Markdown |
| `*.md` | 根目录 Markdown（README 等） |

---

## UI 组件（触发 UI 审查）

| 模式 | 说明 |
|------|------|
| `**/*.tsx` | React/TSX 组件 |
| `**/*.jsx` | React/JSX 组件 |
| `**/*.vue` | Vue 组件 |
| `**/*.svelte` | Svelte 组件 |

---

## 类型定义（触发类型审查）

| 模式 | 说明 |
|------|------|
| `**/types.ts` | 类型定义文件 |
| `**/types/*.ts` | types 目录下的文件 |
| `**/*.d.ts` | 类型声明文件 |
| `**/interfaces.ts` | 接口定义文件 |

---

## 配置文件（触发配置审查）

| 模式 | 说明 |
|------|------|
| `**/config.ts` | 配置文件 |
| `**/*.config.ts` | 配置文件（如 vite.config.ts） |
| `**/constants.ts` | 常量定义 |

---

## 排除模式

以下目录/文件不触发审查：

| 模式 | 说明 |
|------|------|
| `node_modules/**` | 依赖目录 |
| `dist/**` | 构建输出 |
| `build/**` | 构建输出 |
| `.git/**` | Git 目录 |
| `*.min.js` | 压缩文件 |
| `*.map` | Source map |
| `*.lock` | 锁文件 |

---

## 变更类型判定

根据文件路径判定变更类型，用于生成审查提示：

| 变更类型 | 判定条件 | 提示标签 |
|----------|----------|----------|
| 文档 | 匹配 `docs/**/*.md` 或 `*.md` | `[文档]` |
| UI | 匹配 `**/*.tsx` `**/*.vue` 等 | `[UI]` |
| 类型 | 匹配 `**/types.ts` `**/*.d.ts` 等 | `[类型]` |
| 配置 | 匹配 `**/config.ts` `**/constants.ts` 等 | `[配置]` |
